# 🚀 Распределенная система MapReduce (на базе MIT 6.5840)

Данный проект представляет собой отказоустойчивый фреймворк распределенных вычислений MapReduce, написанный на языке Go. Проект вдохновлен оригинальной статьей Google и разработан в рамках изучения архитектуры распределенных систем (курс MIT 6.5840).

Система позволяет параллельно обрабатывать гигантские объемы данных, автоматически распределяя задачи между множеством независимых рабочих процессов (воркеров).

## 🧠 Архитектура и возможности

* **RPC-коммуникация:** Координатор (Master) и Воркеры общаются по сети через Remote Procedure Call (пакет `net/rpc`). Воркеры запрашивают задачи и отчитываются о результатах.
* **Отказоустойчивость (Fault Tolerance):** Координатор не доверяет воркерам. Если воркер берет задачу, но не отвечает в течение 10 секунд (например, отключилось питание или завис процесс), Координатор считает его мертвым и автоматически передает задачу другому свободному воркеру.
* **Безопасность потоков (Concurrency Safety):** Интенсивное использование `sync.Mutex` для защиты структур данных Координатора от состояния гонки (Data Race) при одновременных запросах от десятков воркеров.
* **Атомарная запись:** Использование временных файлов (`temp`) и атомарной операции `os.Rename` на уровне ОС. Это гарантирует, что в случае внезапного падения воркера на диске не останется "битых" или наполовину записанных файлов.
* **Идемпотентность:** Защита от дублирующихся отчетов (например, когда "зависший" воркер внезапно просыпается и пытается сдать работу, которая уже выполнена другим).

## 🗺️ Схема работы

```text
       [ Входные файлы (например, pg-*.txt) ]
                         |
                 +---------------+
                 |  Координатор  | <--- Отслеживает статусы (Idle, InProgress, Completed)
                 +---------------+      Запускает 10-секундные таймеры для каждой задачи
                  /      |      \
         (RPC)   /     (RPC)     \   (RPC)
                /        |        \
      [Воркер 1]    [Воркер 2]    [Воркер N]  <--- Выполняют Map и Reduce
                \        |        /
                 \       |       /
          [ Итоговые файлы (mr-out-0, mr-out-1...) ]
```

## 🛠️ Инструкция по локальному запуску (Ручной режим)

Фреймворк спроектирован так, что пользовательскую логику (например, приложение для подсчета слов) можно подключать в виде скомпилированных плагинов.

Для запуска перейдите в директорию `src/main`:

```bash
cd src/main
```

### Шаг 1: Компиляция плагина

Скомпилируйте приложение подсчета слов (Word Count) в виде динамического плагина:

```bash
go build -buildmode=plugin ../mrapps/wc.go
```

### Шаг 2: Запуск Координатора

Откройте первый терминал и запустите Координатор. Ему нужно передать имя сетевого сокета (например, `mysocket`) и набор текстовых файлов для анализа:

```bash
go run mrcoordinator.go mysocket pg-*.txt
```

Координатор запустится и перейдет в режим ожидания подключений.

### Шаг 3: Запуск Воркеров

Откройте второй терминал (или несколько терминалов, чтобы симулировать кластер машин) и запустите воркер, передав ему скомпилированный плагин и то же самое имя сокета:

```bash
go run mrworker.go wc.so mysocket
```

Результат: Воркеры подключатся к Координатору, заберут Map-задачи, сгенерируют промежуточные файлы, затем выполнят Reduce-задачи. По завершении работы в директории появятся файлы `mr-out-*` с итоговыми результатами вычислений.

## 📚 Чему я научился в этом проекте

* **Проектирование распределенных систем:** Понимание того, как независимые узлы могут работать над одной задачей без единой точки отказа.
* **Продвинутый Go:** Уверенное владение горутинами (Goroutines), каналами и мьютексами.
* **Клиент-серверное взаимодействие:** Создание надежных сетевых контрактов с использованием `net/rpc`.
* **Отладка многопоточного кода:** Устранение дедлоков (deadlocks) и гонок данных (data races) в условиях высокой конкурентности.
